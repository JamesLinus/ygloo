Cancel all requests that match a given tag, It cancels pending requests immediately, but active requests only if the remaining bytes to transfer exceeds the threshold bytes, up to a hard timeout.

Parameters
httppool
tag
thresholdbytesIf a positive number, then the number of bytes permitted to complete. If 0, then the timeout is used alone. If -1, then cancellation is immediate.
timeoutmsThe max time allowed for the request to complete prior to cancelling, in milliseconds. If 0, then cancellation will happen based on thresholdbytes. If -1, then cancellation is immediate.
Returns
-1 on error or number of cancelled requests

int httppool_setkeepalivepoolsize ( HttpPool *  httppool,
int  keepalivepool

Set the maximum number of cached connections that are being kept alive. You should aim for a number that allows you to cache several connections to hosts that are being called concurrently. For example if your pool connects to 10 different hosts you should use at least a pool size of 30 to cache 3 connections per host.

Set the maximum number of concurrent requests. If pipelining is disabled, the actual number of concurrent requests will be limited by the maximum number of concurrent connections. But if pipelining is enabled, then multiple requests can share a connection, and the number of concurrent requests can exceed the number of concurrect connections. The default is 50.
