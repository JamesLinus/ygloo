Ymagine

Image decoding and processing library

Download this project as a .zip file Download this project as a tar.gz file

Overview

Ymagine is a portable, highly efficient and light weight image decoding and processing library. On Android platform, Ymagine provides a re-implementation, on top of the native API, of BitmapFactory, as usually provided by Android framework. It supports extended capabilities to allow reusing Bitmap, apply resizing and shaders transformation on the fly in a highly memory and speed efficient way when decoding images, and consistent set of image formats and features across all Android platform levels.

Getting started

Downloading the source

Installing repo

Repo is a tool that makes it easier to work with Git for project involving a large collection of repositories. It was created and is mostly used to manage Android Open-Source codebase. For more information about Repo, see the Developing section of Android project.

Ygloo is built on top of some open-source projects, and combines several projects into one platform. as such, Repo provides a convenient way to manage this collection of repositories.

To install Repo:

  1. Make sure you have a bin/ directory in your home directory and that it is included in your path:

    mkdir ~/bin
PATH=~/bin:$PATH

  2. Download the Repo tool and ensure that it is executable:

    curl https://storage.googleapis.com/git-repo-downloads/repo > ~/bin/repo
chmod a+x ~/bin/repo

Initializing repo client

After installing Repo, set up your client to access the Android source repository:

  1. Create an empty directory to hold your working files. If you're using MacOS, this has to be on a case-sensitive filesystem. Give it any name you like:

    mkdir WORKING_DIRECTORY
cd WORKING_DIRECTORY

  2. Run repo init to bring down the latest version of Repo with all its most recent bug fixes. You must specify a URL for the manifest, which specifies where the various repositories included in the Android source will be placed within your working directory.

    repo init -u git@github.com:yahoo/ygloo.git

    To check out a branch other than "master", specify it with -b:

    repo init -u git@github.com:yahoo/ygloo.git -b v0.5.0

  3. When prompted, configure Repo with your real name and email address. To use the Gerrit code-review tool, you will need an email address that is connected with a registered Google account. Make sure this is a live address at which you can receive messages. The name that you provide here will show up in attributions for your code submissions.

A successful initialization will end with a message stating that Repo is initialized in your working directory. Your client directory should now contain a .repo directory where files such as the manifest will be kept.

Downloading source tree

To pull down the Ygloo source tree to your working directory from the repositories as specified in the default manifest, run

repo sync

The Ygloo Native SDK source files and build system will be located under their project names. For more about repo sync and other Repo commands, see the Developing section of Android documentation.
